# 胜平负方向冲突根因审计（2026-07-23）

## 审计边界

本轮只分析既有锁版记录和不可变模型运行快照，不修改模型、权重、评级、门禁、锁版或
生产数据。

- 审计记录：198 场。
- 严格概率样本：112 场。
- 模型与市场方向一致：87 场。
- 模型与市场方向冲突：25 场。
- 其中可关联完整模型运行快照：22 场。
- 缺运行快照、只能做结果审计：3 场。

## 核心结论

25 场方向冲突中，模型命中 8 场，市场命中 13 场，双方都未命中 4 场。22 场有运行
快照的冲突，原始 Poisson/xG 方向全部已经与市场方向相反。

| 冲突来源 | 场次 |
|---|---:|
| 原始 Poisson/xG 源已经反向 | 22 |
| 比分历史/比分市场混合首次造成反向 | 0 |
| 85/15 胜平负混合首次造成反向 | 0 |
| 赛前证据修正首次造成反向 | 0 |
| 缺运行快照 | 3 |

22 场中只有 1 场在原始 Poisson 与最终联合比分分布之间换了另一个方向；其余 21 场
方向没有变化。也就是说，比分历史、比分市场、85/15 胜平负混合和研究证据主要是在
继承一个已经反向的 xG 底盘，而不是首次制造问题。

这比“联合比分模型弱于市场”更进一步：当前可追溯冲突的共同起点是
`expectedGoals()` 生成的主客 xG 关系。

## 全部 91 场运行快照的源层表现

不只看冲突场，91 场可拆解运行快照的结果如下：

| 概率阶段 | 命中 |
|---|---:|
| 原始 Poisson/xG 方向 | 40 / 91（44.0%） |
| 联合比分分布方向 | 44 / 91（48.4%） |
| 市场去水方向 | 50 / 91（54.9%） |
| 最终模型方向 | 46 / 91（50.5%） |

比分历史与比分市场混合从 40 场修复到 44 场，但仍未超过市场。后续证据修正再净增
2 场，也没有消除源层劣势。

## xG 输入层的可验证风险信号

以下计数来自 22 场有运行快照的冲突。标签允许重叠，只用于定位风险，不等同于已经
证明单一因果。

| 风险信号 | 冲突场次 |
|---|---:|
| 近期赛果攻防方差至少一项 ≥ 2 | 11 |
| 主客 xG 差 ≥ 0.75 | 10 |
| 至少一队直接主/客场样本少于 3 | 7 |
| 至少一队近期有效样本少于 5 | 4 |
| 运行快照内存在重复近期赛果 | 4 |

22 场里有 20 场至少命中一个上述风险信号。阈值是诊断阈值，不应直接改造成新的正式
门禁；它们说明当前 xG 估计没有充分处理样本量、方差、重复记录和球队身份质量。

对全部 91 场运行快照继续分组：

| 分组 | 市场命中 | 联合比分命中 |
|---|---:|---:|
| 至少一队近期样本少于 3（5 场） | 4 / 5 | 2 / 5 |
| xG 差 ≥ 0.75（39 场） | 27 / 39 | 19 / 39 |
| 攻防方差至少一项 ≥ 2（32 场） | 16 / 32 | 12 / 32 |
| 直接主/客场样本少于 3（17 场） | 8 / 17 | 6 / 17 |

最值得警惕的是 xG 差值：模型拉开到 0.75 以上时并没有更可靠，反而比市场少命中
8 场。这说明大幅 xG 关系当前不是“高置信证据”。

## 已确认的具体失败方式

### 1. 极少量样本仍生成高幅度 xG

- 奥莫尼 vs 阿拉木：近期有效样本为 0 / 1，xG 为 0.81 / 1.93；市场主胜、赛果主胜，
  模型客胜。
- 洛杉矶 vs 皇家盐湖：近期有效样本为 1 / 7，xG 为 1.20 / 2.44；市场主胜、赛果
  主胜，模型客胜。`洛杉矶`还是城市级短名，不能稳定区分 LAFC 与洛杉矶银河。
- 格拉茨 vs 哈茨：进入 xG 的有效近期样本为 2 / 0，xG 为 1.42 / 2.50；市场主胜、
  赛果主胜，模型客胜。

这些场次虽然最终可能是 `PRE_LOCK`，但它们仍证明基础概率会在证据不足时输出方向性
很强的数值。门禁只阻止发布，不能让概率本身变正确。

### 2. 三场直接主客样本即可主导 xG，且方差没有用于收缩

`expectedGoals()` 在某队直接主场或客场样本达到 3 场时，就完全改用这些直接场地
样本。攻防方差只被记录到审计字段，没有进入 xG 收缩。

- 汉坎 vs 特罗姆：xG 2.54 / 0.30，最大方差 4.67；市场客胜、赛果客胜，模型主胜。
- TPS vs 坦山猫：xG 3.38 / 1.41；市场客胜、赛果客胜，模型主胜。
- 索尔纳 vs GAIS：xG 1.13 / 2.32；市场主胜、赛果主胜，模型客胜。

当前算法把少量主客场大比分直接平均，再与对手相反侧防守均值相加，容易被 4-0、5-0、
0-5 等单场极端值放大。

### 3. 同联赛样本不做对手强度校正

跨联赛样本要求联赛强度、对手质量和比赛类型完整；同联赛样本的
`strengthFactor` 固定为 1。于是同联赛内对强队和弱队的进失球被视为等价。

当前 xG 公式本质是：

```text
主队 xG = (主队近期进球均值 + 客队近期失球均值) / 2 × 联赛开放度
客队 xG = (客队近期进球均值 + 主队近期失球均值) / 2 × 联赛开放度
```

它没有对同联赛对手强弱、升班马、赛程强度和样本不确定性进行收缩。这是结构性限制，
不是增加一个评级门槛能修复的问题。

### 4. 市场强方向可以被很小的 xG 差覆盖

利勒斯 vs 维京的 xG 只有 1.63 / 1.50，差值 0.13，但原始 Poisson 仍给出主胜方向，
并通过 85% 联合比分权重压过市场客胜；最终赛果为客胜。

这说明问题不仅存在于极端 xG。只要模型与市场冲突，固定 85% 权重就默认模型拥有已经
被证明的独立优势，而历史审计恰好不支持这一前提。

## 版本和联赛信号

- R15 的 11 场最终方向冲突：市场赢 6、模型赢 2、双方都错 3。
- R16 的 3 场最终方向冲突：市场 3 场全赢，模型 3 场全错。
- 韩职 6 场冲突中模型赢 4、市场赢 1、双方都错 1。
- 瑞超、挪超、欧冠、芬超和美职的冲突合计中，市场明显占优。

韩职可能存在可学习的模型残差，但只有 6 场冲突，不能据此直接建立联赛特调；应作为
后续时间外验证假设。R15/R16 的最新冲突结果则足以否定“当前通用 xG 可以默认覆盖
市场方向”。

## 下一步应验证什么

本轮不修改模型。下一轮模型重建应按以下顺序做离线 Challenger，不直接碰 Champion：

1. 固化球队身份映射，拒绝城市级歧义短名进入球队近期状态。
2. 对本地样本、滚动样本和 Case Base 做比赛级去重，并把重复率纳入训练数据报告。
3. 用分层/收缩估计替代“3 场即完全采用主客场均值”；方差和样本量必须进入估计，不是
   只进入门禁。
4. 同联赛样本加入对手强度和赛程强度校正。
5. 将市场去水概率作为先验，球队状态只学习相对市场的残差；任何主动反向必须在时间外
   样本证明有增量。
6. 用锁版时间滚动验证方向命中、Brier、Log Loss 和冲突场表现，禁止随机切分和未来
   数据泄漏。
