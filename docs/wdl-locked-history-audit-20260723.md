# 胜平负锁版历史审计（2026-07-23）

## 结论

本轮不以增加门禁或减少正式放行为目标，而是检验胜平负概率源本身。

审计结果确认：当前主要失效发生在“联合比分模型 → 胜平负概率”这一层。现行固定的
`85% 联合比分模型 + 15% 体彩市场概率` 混合关系没有得到历史锁版记录支持；在当前
严格样本中，它显著弱于市场概率基线。赛前证据修正只挽回极少数方向，没有修复基础
概率源的整体劣势。

因此，在重新证明联合比分模型具备独立增益前，不应继续提高它在胜平负概率中的权重，
也不应通过继续收紧 A/B 评级或正式门禁来掩盖概率层问题。

## 样本边界

- 审计记录：198 场。
- 严格概率回测样本：112 场。
- 仅可做结果审计、不能做严格概率校准：86 场。
- 严格样本要求：
  - 每场只保留最新人工锁版，避免同一比赛重复计数；
  - 锁版时间早于开赛时间；
  - 有不可变赛前模型胜平负概率；
  - 有赛前普通胜平负 SP 或去水市场概率；
  - 有 90 分钟正式赛果。

主要缺口不是“没有历史记录”，而是早期记录缺不可变赛前时间或普通胜平负市场快照。
所以 198 场足够定位全局故障，也足够验证低参数校准；但不能把缺字段的 86 场伪装成
完整概率训练样本。

## 总体结果

| 概率源 | 命中 | Brier | Log Loss |
|---|---:|---:|---:|
| 已锁版模型概率 | 61 / 112（54.46%） | 0.5738 | 0.9666 |
| 同场市场去水概率 | 66 / 112（58.93%） | 0.5352 | 0.9108 |

三个指标方向一致：当前模型不仅方向命中低于市场，概率质量也低于市场。

## 失效发生在哪一层

91 场同时保留完整模型运行快照，可以进一步拆解概率链路：

| 计算阶段 | 命中 | Brier | Log Loss |
|---|---:|---:|---:|
| 联合比分模型原始概率 | 44 / 91（48.35%） | 0.6153 | 1.0259 |
| 市场去水概率 | 50 / 91（54.95%） | 0.5600 | 0.9465 |
| 固定 85/15 混合后 | 46 / 91（50.55%） | 0.6030 | 1.0078 |
| 赛前证据修正后 | 46 / 91（50.55%） | 0.6032 | 1.0084 |

这组拆解排除了两个错误解释：

1. 不是后续评级门禁造成胜平负概率变差。概率在进入评级前已经弱于市场。
2. 不是证据修正把一个正确底盘改坏。证据修正前后的总体命中相同，概率损失也几乎
   没有变化。

真正的问题是：联合比分模型当前没有提供稳定增量，却占了胜平负底盘 85% 权重。

## 时间滚动样本外验证

按锁版时间排序，前 40 场只用于拟合，后续每 10 场向前滚动，禁止用未来赛果修改过去
参数。共得到 51 场严格样本外结果：

| 方案 | 命中 | Brier | Log Loss |
|---|---:|---:|---:|
| 当前固定 85/15 | 22 / 51（43.14%） | 0.6358 | 1.0534 |
| 滚动学习单一混合权重 | 29 / 51（56.86%） | 0.5665 | 0.9555 |
| 市场概率基线 | 29 / 51（56.86%） | 0.5557 | 0.9379 |
| 联合比分模型 | 21 / 51（41.18%） | 0.6557 | 1.0837 |

混合权重最初拟合为联合比分 59%、市场 41%；从第 50 个训练样本开始，之后五个滚动
窗口都收敛为联合比分 0%、市场 100%。这不是主张永久照搬市场，而是说明现有历史
证据不支持联合比分模型继续主导胜平负概率。

## 版本与分歧诊断

- R15：模型 17/40（42.5%），市场 21/40（52.5%）。
- R16 当前已结算严格样本：模型 4/9（44.4%），市场 7/9（77.8%）。样本仍小，只能
  视为强烈预警，不能单独作为最终结论。
- 模型和市场同方向时：模型方向 53/87（60.9%）。
- 模型和市场冲突时：模型 8/25（32.0%），市场 13/25（52.0%）。

模型的主要损失集中在“主动偏离市场”的比赛，而不是市场与模型一致的比赛。

## 关于平局

112 场实际平局占 22.32%，模型平均给出的平局概率为 24.53%，不存在全局平均意义上的
“平局概率被清零”。但模型只在 4/112 场把平局排在第一位，市场则是 0/112。

因此，“7 月 7 日后几乎没有平局结论”需要分成两个问题：

1. 概率问题：平均平局概率并未整体缺失。
2. 决策问题：当前单选使用三项概率最大值，结构上会极少选择平局。

不能仅凭平局结论少就强行增加平局输出；下一步应检验哪些赛前特征能让平局相对主胜或
客胜获得可重复的条件优势，并用样本外数据验证。

## 当前应执行的模型修正方向

1. 保留 R16 为 Champion，仅用于确保线上输出稳定，不再继续增加门禁来“优化命中率”。
2. 将“市场去水概率 + 时间温度校准”作为概率源 Challenger。刷新后的 72 场滚动样本外
   结果为 43/72（59.72%），Brier 0.5285，Log Loss 0.8974；同期原模型为
   36/72（50.00%），Brier 0.5973，Log Loss 1.0012。
3. 联合比分模型暂时保留为比分、总进球和让球的候选底盘；它若要重新进入胜平负主概率，
   必须在新的时间外样本中证明相对市场的增量，而不是预设 85% 权重。
4. R17/R18 继续以同输入、不可变赛前快照做前向配对。达到既定样本数并同时通过方向、
   Brier、Log Loss、覆盖率和等级单调性后，再人工决定是否晋级。

## 可重复执行

训练集构建支持附带模型运行快照：

```bash
node tools/build-wdl-training-set.mjs \
  --locks /tmp/locks.json \
  --runs /tmp/model-runs.json \
  --results /tmp/live-results.js \
  --static-data web/data.js \
  --output /tmp/wdl-training.json
```

执行历史审计：

```bash
npm run wdl:audit-history -- \
  --input /tmp/wdl-training.json \
  --output /tmp/wdl-history-audit.json
```

该审计只读取锁版快照和赛果，不修改 Champion、锁版、Case Base 或生产数据库。
