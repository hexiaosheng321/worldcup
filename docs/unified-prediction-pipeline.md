# 统一赛前推演流程

所有世界杯、芬超、韩职、瑞超、挪超比赛统一使用稳定主版本 `UNIFIED_PREDICTION_V4`。旧的联赛生成器也只调用同一个引擎，不再各自维护概率、比分和门禁逻辑。旧锁版永久保留原版本，新规则不得覆盖历史锁版。

## 稳定主版本契约

- 胜平负、让球、总进球和比分共享同一个联合比分概率分布；各玩法只做独立汇总和有限校准。
- 体彩胜平负、让球、比分和总进球赔率只做校准，权重上限分别为 15%、10%、10%、15%，不能替代基本面。
- 每队至少需要 5 场新鲜赛前比赛数据；状态不足必须进入 `DATA_REPAIR`，不得把市场权重放大后强行锁版。
- 伤停和预计首发必须先外部搜索。可靠信息尚未发布时记录 `NOT_PUBLISHED`、来源和搜索时间，量化影响强制为0并降低数据质量；它们不是单独否决锁版的决定性因素。
- 只有锁版前已存在的数据可以参与推演，所有证据保存 `observedAt`、`capturedAt` 和来源。
- 胜平负和让球先各自选出概率第一项。若两项无共同比分，一致性检查只负责触发冲突；胜平负主方向保持不变，正式让球按该方向下完整联合比分分布选择，条件 Challenger 记录排除正式项后的次优分支。它们都不能为了让球兼容而反转胜平负主方向。
- 生命周期固定为 `DATA_PENDING → DATA_REPAIR → MODEL_READY → CONSISTENCY_CHECK → FINAL_LOCK → RESULT_SETTLED → BASE_CASE`。
- 当前线上版本是 Champion；任何新权重先以 Challenger 影子回测，样本外 Brier Score、Log Loss 和校准率稳定优于主版本后才允许升级。
- 自学习按“联赛 × 当前赛季”隔离：持续统计近期主胜率、平局率、客胜率、场均进球和双方进球率相对赛季基线的变化，形成量化赛季叙事。新叙事默认只进入 `CHALLENGER_SHADOW`，不得自动改写 Champion。
- Challenger 晋级的第一目标是四项结果质量：胜平负单选、让球单选、总进球双选、比分双选；同时要求样本外 Brier Score、Log Loss 和校准率不退化，禁止只凭单场赛果升级。
- 只有 preferred `FINAL_LOCK` 能在90分钟赛果结算后生成 Base Case，下一次推演同时读取成功和失败案例。
- 两个正式比分只服务联赛、赛季校准后的最大联合概率覆盖；独立风险路径另列，并依其方向概率与单一比分概率对最终置信度扣分，不占正式比分名额。
- 胜平负只由完整联合比分分布、有限市场校准和赛前证据生成。两个正式比分是下游最大覆盖输出，只作比分覆盖与玩法兼容审计，不再归一化后重复回灌胜平负。当市场、首球方和两回合追分暴露至少两项支持独立风险方向时，只进入35% Challenger影子审计，未满30至50场且样本外指标未验证前不得改写Champion。
- 两回合赛事必须提供结构化 `tieContext`，分别计算90分钟胜平负、本场净胜球差和总比分晋级状态；只有自然语言而没有回合数及赛前总比分时，阻断 `FINAL_LOCK`。
- 正式胜平负使用方向概率第一项；正式让球在该方向下按完整联合比分分布的净胜球概率选择。两个最大覆盖比分只作支持性验证：若不支持正式让球，必须保留 `PRE_LOCK / 跳过`，不得用单个 `1-0` 或 `2-1` 反向决定让球单选。禁止把彼此互斥的两个独立概率第一项直接组成正式输出。
- 让球独立边际第一项保留为风险审计；主方向下排除正式 Champion 后的次优净胜球分支保留为条件 Challenger，禁止两条轨道重复。正式让球、独立边际、条件 Challenger 与胜平负+让球联合命中必须分轨统计。
- 正式让球若偏离独立边际概率第一项超过10个百分点，仍输出逻辑兼容的完整 `PRE_LOCK / 跳过`，但不得升级 `FINAL_LOCK`。
- 联赛强弱差的 xG 必须优先使用主队主场和客队客场样本（各至少3场），记录攻防方差，并按该联赛近期场均总进球得到的开放度系数校正；样本不足才回退到总体近况。
- 跨联赛近期赛果进入 xG 前，必须同时提供联赛强度、对手质量、正式赛/友谊赛类型和样本时间；缺失归一化因子的跨联赛样本不得进入计算。
- 当锁版市场方向、风格层首球方、两回合追分暴露至少两项支持同一反向路径，而数值主方向相反时，必须有两项独立量化证据解释，否则阻断 `FINAL_LOCK`。
- 两回合领先方缺少净胜球需求、对手持续威胁或体能崩盘证据时，对领先方后续进球 xG 执行12%衰减，避免把“能够取胜”自动放大为“三球以上”。
- 体彩赛事阶段、研究阶段和结构化两回合状态必须一致；标签缺失或互相冲突时阻断 `FINAL_LOCK`。

## 每场比赛的固定顺序

1. 从线上 D1 bootstrap 读取当前比赛和体彩胜平负、让球赔率；`full` 失败时自动降级到 `initial`。
2. 读取线上 SP 历史和 D1 滚动历史样本，计算去水市场概率、近期状态、Poisson 比分和两套情景。
3. 生成七项赛前资料模板：球队状态、伤停、预计首发、动机、天气场地、风格对位、市场消息。
4. 使用搜索逐项补齐，并为每项保存摘要、抓取时间、事实发生/更新时间 `observedAt`、来源 URL、证据等级和数值影响。无法核实的项目保持 `MISSING`，禁止推断成已确认事实。
5. 统一执行赔率完整性、样本量、盘口变化、赛前搜索、比分校验、冲突处理和让球映射七道门禁。
6. 10步中的每一步输出 0-100 分与通过状态；任一道门禁失败只能输出 `PRE_LOCK`，全部通过才允许 `FINAL_LOCK`。
7. 使用 `--publish-run true` 将本次完整可回放输入与输出写入 D1 `model_runs`：保留原始四市场赔率、SP历史、赛前研究、同联赛样本与两队跨联赛归一化样本，禁止只保存输出后的概率摘要。正式锁版必须携带该 `modelRunId`，API 会验证比赛、10步门禁、主比分和让球映射，不能再以 `finalApproval` 单独绕过。
8. 赛后复盘记录 Brier Score、Log Loss 和概率校准分箱，除命中率外同时评价概率质量。
9. `跳过/VOID` 只从正式投注命中率中排除，仍必须验票胜平负、让球、总进球和比分四个组件，并作为影子样本进入联赛与赛季校准；任何组件失败都不得生成“命中样本沉淀”。

```bash
npm run prediction:unified -- --match 1320350 --lock FINAL_LOCK
npm run prediction:unified -- --match 1320350 --evidence /tmp/research-1320350.json --lock FINAL_LOCK --publish-run true
```

资料模板默认写入 `/tmp/unified-research-template.json`。预计首发和临场伤停尚未发布时，应先保留 `PRE_LOCK`，开赛前再次搜索补齐后重跑。

每项研究证据的 `impact` 均为有上限的概率/xG修正，格式如下；中性信息必须明确填写 0，不能省略：

```json
{
  "status": "VERIFIED",
  "evidenceGrade": "A",
  "summary": "可复核的赛前事实与判断依据",
  "capturedAt": "2026-07-10T08:00:00.000Z",
  "observedAt": "2026-07-10T07:30:00.000Z",
  "sources": [{ "title": "来源", "url": "https://example.com" }],
  "impact": { "home": 0.02, "draw": 0.01, "away": -0.03, "xgHome": 0.1, "xgAway": -0.1 }
}
```

两回合赛事的资料文件还必须包含：

```json
{
  "competitionStage": "QUALIFYING",
  "tieContext": {
    "isTwoLeg": true,
    "legNumber": 2,
    "aggregateHomeBeforeMatch": 0,
    "aggregateAwayBeforeMatch": 1,
    "leaderNeedsGoalDifference": false,
    "trailingSideSustainedThreat": false,
    "trailingSideCollapseRisk": false
  }
}
```

跨联赛的 `teamState.recentMatches` 每场还必须补齐 `leagueStrengthFactor`、`opponentQualityFactor`、`matchType` 与比赛时间。友谊赛权重自动降为45%，其余样本继续按时效衰减。

## 2026-07-10 连续失败复盘规则

- 让球不穿只调整让球结论，不得自动把胜平负热门改成平局。
- 最终胜平负按全部场景概率汇总，不能由单个主比分直接决定。
- 两个正式比分必须选择联赛、赛季校准后联合概率最高的两个落点，允许覆盖同一赛果方向；独立风险剧本另列，不占正式比分名额。
- 当市场第一方向领先平局至少 12 个百分点时，改选平局必须有至少两项独立量化证据。
- 球队状态/风格最多使用 7 天内信息，伤停/首发最多 24 小时，天气最多 48 小时，市场消息最多 6 小时。
- `PRE_LOCK` 只作影子测试，不进入正式命中率；同一比赛只有 preferred `FINAL_LOCK` 可以生成正式 Case。

## 2026-07-12 Base Case 学习规则

- 如果赛后真实比分命中锁版的第二正式比分，复盘必须标记“第二概率落点命中”；独立风险剧本单独验票，并只进入风险诊断和置信扣分。
- 如果实际净胜球显著偏离预测，复盘必须区分“强队幅度低估”和“强队幅度高估”；下次读取同联赛 Base Case 时，用主客场攻防方差和联赛开放度调整比分分布的尾部，不再只用进球均值。

## 2026-07-16 三场复盘优先规则

- P0：欧冠资格赛等跨联赛对阵必须先做实力归一化，不能直接比较国内联赛原始进失球。
- P0：市场、首球判断和追分暴露形成二对一反向证据时，两项独立量化解释不足就降为 `PRE_LOCK`。
- P1：证据驱动的风险路径以35% Challenger影子权重验票，不机械提高第二正式比分权重。
- P1：两回合领先方无继续扩大比分证据时衰减后续进球，分别判断胜负方向与进球幅度。
- P1：赛事阶段标签、研究上下文和两回合结构不一致时禁止锁版。
- D级跳过场继续影子验票；至少积累30至50场后再决定是否调整出手阈值。

## 2026-07-16 R10 学习闭环

- 正式让球按胜平负主方向下的完整联合净胜球分布选择；两个正式比分只作硬验证，不再由单个 `1-0` / `2-1` 决定让球单选。
- 每个锁版必须携带 `modelRevision`、`scoreSelectionPolicy` 和完整 `backtestContract`，复盘按版本分组，禁止把 R10 与历史版本混合宣称命中率提升。
- 让球赛后分别记录正式单选、独立边际第一项、主方向下次优条件 Challenger 和胜平负+让球联合命中，不得只留一个 `handicapHit`。
- 同场最新 `PRE_LOCK` 赛后写入独立 `shadow_model_audits`，不进入正式 Case Base，但必须生成四组件与 Challenger 分轨验票。
- 复盘失败只能生成 `SHADOW_PENDING` Challenger；同联赛、同赛季、同版本累计30至50场，目标命中率提高且联合命中、Brier Score、Log Loss不退化后，才可人工采纳为 Champion。

## 2026-07-16 R11 胜平负去重复回灌

- 完整联合比分网格已经进入胜平负基线，正式选出的两个比分不得再次被归一化为100%后以固定权重回灌方向。
- `scenarioDirectionCalibration` 为兼容旧消费者保留，但固定 `weight=0`、`applied=false`，仅作为正式比分方向覆盖审计。
- 两个正式比分仍参与比分覆盖、总进球覆盖、让球兼容门禁和整体置信覆盖贡献，但不得改变胜平负概率第一项。

## 2026-07-17 R12 组件门禁与分轨建议

- 继承 R11 的胜平负方向，不根据本轮赛果重写方向概率；今日复盘只升级资格赛样本、让球幅度、总进球一致性和组件建议。
- 资格赛任一方有效主客场样本为0时阻断 `FINAL_LOCK`；只有补齐至少3场、强度因子在0.85至1.15之间并带可复核来源的同强度替代样本才允许继续。
- `-1` 主让且第一正式比分为 `1-0` 或 `2-1` 时，主胜不得自动升级成让胜；模型保留让球候选，但将正式让球降为谨慎并阻断整包锁版。
- 总进球双选先覆盖两个正式比分对应的总球数，再检查与 xG 总和、完整比分分布的比赛类型是否一致；高开放度、高方差、强方向场还必须覆盖5球以上尾部，否则保持 `PRE_LOCK`。
- 胜平负、让球、总进球和比分分别输出概率、ABCD等级、可出手状态及 `主打/可选/谨慎/跳过`，不再让整场评级被胜平负单项带高；关键样本、时效或研究基础不完整时四组件统一封顶D且不得成为正式玩法。
- 整包 `overallGrade` 先检查共享证据基础，再按可出手组件广度和强组件数量评级：关键基础缺失或没有强可出手组件为D；只有单项可靠或部分组件冲突为C；四项都可用且至少两项A/B才可到B；四项全部A才可到A。
- 上述规则先作为通用门禁和组件分轨验票；挪超打花、欧联资格赛、巴西甲一球胜和美职状态方差只记录为联赛触发场景，未满30至50场前不扩大联赛概率权重。

## 2026-07-17 R13 等级与建议动作对齐

- 整包等级不再机械取四组件最低值，避免把“单项方向可用、衍生玩法冲突”和“共享基础证据缺失”混成同一种D级。
- A级整包才允许进入主打候选；B级整包建议最高为可选；C级整包建议最高为谨慎；D级保持跳过。组件建议仍独立保留，因此胜平负A级可以单项主打，但不能带高整包评级。
- `overallGradeAudit` 必须记录基础是否有效、可用组件数、可出手组件数、A/B强组件数和评级原因，赛后按等级与覆盖率共同回测。

## 2026-07-17 R14 输出一致性评分与关键缺口D级封顶

- `outputConsistencyScore` 为0至100分：总进球双选完整度10分、与xG距离25分、两个正式比分总球覆盖25分、比赛类型一致20分、高方差尾部覆盖20分；评分与命中率按联赛、赛季、模型版本共同回测。
- 一般不一致只降低一致性分和置信度，可保留C级单项观察；总进球选择不完整、正式比分总球未覆盖、5球以上高方差尾部缺失、xG严重偏离或一致性低于60分才定义为严重输出冲突。
- 关键样本/证据基础、资格赛场地样本、两回合/赛事阶段、`让胜 ↔ 让负` 极端让球冲突或严重输出冲突任一成立时，整包强制D级、`PRE_LOCK`、建议跳过并清空 `formalMarkets`。
- 关键缺口场仍保留四组件等级与 `observationalMarkets`，只进入影子验票；不得因为胜平负单项看似可靠而恢复整包正式推荐。
- 评分阈值先固定进入30至50场样本外回测，不根据单场命中调整；晋级要求正式命中率、覆盖率、Brier Score和Log Loss同时不退化。

## 2026-07-17 R15 共享缺口与玩法局部缺口分层

- 保留R14的输出一致性评分和四玩法独立评级，不调整ABCD概率阈值；版本升级只改变关键缺口的作用范围，避免同一版本号对应两套门禁逻辑。
- 共享证据基础、资格赛主客场有效样本、两回合状态或赛事阶段存在严重缺口时，继续执行整包D级、`PRE_LOCK`、跳过并清空全部 `formalMarkets`。
- `让胜 ↔ 让负` 极端冲突属于让球局部缺口，只关闭 `handicap`；严重输出一致性冲突属于总进球与比分局部缺口，只关闭 `totalGoals` 和 `scores`。任一局部缺口仍阻断整包 `FINAL_LOCK`，但不得清空不依赖该缺口且自身评级合格的玩法。
- 正式玩法只准入A/B级组件；C级保持 `observationalMarkets` 观察与赛后验票，不进入 `formalMarkets`。这只调整等级动作，不改变R14既有ABCD概率阈值。
- `criticalPackageGap` 必须分别记录 `packageBlocking`、`marketBlocking`、`sharedReasons`、`marketReasons` 和 `blockedMarkets`；`overallGradeAudit` 只在共享缺口时强制整包D级，玩法局部缺口由分项等级和正式玩法名单表达。
- 发布载荷同时保留 `candidateSelections` 和 `formalSelections`：前者用于影子验票，后者只包含 `formalMarkets` 已放行的玩法；发布前必须拒绝共享缺口或局部禁用玩法泄漏进正式结论。
- 近7日历史回放只能用于验证门禁作用范围，不能作为R15样本外晋级依据；正式效果必须从完整R15赛前快照开始连续记录30至50场，并同时报告命中率和覆盖率。
- 普通胜平负三项全空、让球胜平负三项完整且至少存在两个同盘口有效快照时，R15 标记 `HHAD_ONLY`：胜平负方向仍由完整联合比分分布和基本面生成，不伪造普通 SP；赔率动态改用让球 SP，并固定扣减4分置信度。普通胜平负只缺一至两项时仍为 `DATA_INCOMPLETE`，不得旁路。
