# Football Decision Engine 今日产品更新

> 2026-07-03 更新版。本文用于官网介绍、社媒发布和外部用户理解项目进展。

## 一句话介绍

我们正在把一个世界杯专题页面，升级成一个可长期积累样本、复盘模型、覆盖多联赛的足球比赛决策引擎。

它不是简单给出“看好谁赢”，而是把赛程、盘口、赔率、历史样本、模型推演、赛前锁版和赛后复盘放进同一条链路里。

## 今天主要优化

### 1. 外部历史样本大幅补齐

今天重点补齐了外部历史样本库，尤其是五大联赛和世界杯相关样本。

当前系统已经不只依赖我们自己的锁版样本，也可以接入外部历史赛果、赔率和盘口数据，用于校验：

- 同联赛胜平负分布
- 常见比分结构
- 总进球区间
- 盘口低位和结果之间的关系
- 相似比赛的赛果分布

这些外部样本不会直接被当成模型命中率，也不会伪装成我们的赛前锁版记录。它们的作用是做分布校验和样本补强，帮助模型判断“这个盘口和比赛形态在历史上通常长什么样”。

### 2. 相似案例逻辑修正

之前相似案例表里存在两个问题：

- 同一场比赛可能重复展示。
- 瑞超等联赛可能误用了世界杯历史样本。

今天已经修正：

- 同一场样本按联赛、日期、主客队和比分统一去重。
- 当前比赛优先使用自身联赛识别，避免因为模型文本里残留“世界杯 V4 经验”而串到世界杯样本。
- 外部样本和锁版样本分清来源，避免把外部历史结果误当成模型成功案例。

这一步很关键，因为相似案例不是装饰信息，而是模型复盘和下一场判断的依据。如果样本来源错了，相似度越高反而越危险。

### 3. 多联赛模型名称统一

今天统一了模型展示名称。

现在规则更清楚：

- 世界杯显示为：`世界杯 V4 模型`
- 其它联赛显示为：`瑞超联赛 V1`、`芬超联赛 V1`、`韩职联赛 V1`

这样做的目的，是避免用户看到类似“瑞超 V1 联赛模型（吸收世界杯 V4 经验）”这种过长表达后误解：瑞超不是世界杯模型，瑞超是自己的联赛 V1，只是复用了世界杯 V4 已经沉淀出来的决策流程。

### 4. 赛事详情页入口统一

今天修复了部分赛事卡片点不进详情页的问题。

现在原则是：

> 只要赛事进入网站，不论是世界杯、瑞超、韩职，还是后续其它联赛，都应该可以点击进入详情页。

详情页会根据状态分成两种：

- 已有真实模型推演：展示锁版结论、盘口依据、比分预测、风险提示和复盘入口。
- 尚未推演或未锁版：展示 `待推演 / 待锁版`，同时保留盘口、联赛画像和数据支持。

这避免了用户看到赛事卡片却无法进入详情的断层，也让网站从“世界杯专题页”更接近“全赛事模型工作台”。

### 5. 首屏加载和数据结构继续优化

今天还继续优化了初始加载和样本拆分。

五大联赛样本量较大，如果全部塞进一个文件，会影响页面加载。现在样本已经开始按联赛拆分，页面加载更可控，也方便后续继续扩展更多联赛。

## 今天更新后的核心价值

今天的更新不是只修几个页面 bug，而是让系统往“长期模型资产”更进一步。

### 从单场分析，走向样本驱动

单场分析很容易变成主观判断。

样本库补齐以后，每场比赛都可以被放回历史分布里看：

- 这个赔率结构历史上冷门多不多？
- 这个让球线是不是容易卡盘？
- 这个联赛常见比分是不是偏开放？
- 当前模型判断是否和历史样本冲突？

这会让模型不只是“写得像分析”，而是真正能被历史数据约束。

### 从世界杯专题，走向多联赛模型

世界杯当前使用 V4。

新增联赛不直接叫 V4，而是从各自 V1 开始：

- 瑞超联赛 V1
- 芬超联赛 V1
- 韩职联赛 V1
- 后续其它联赛 V1

这样既保留世界杯 V4 的成熟流程，又不会假装每个联赛都已经有同等深度的独立验证。

### 从展示页，走向决策闭环

现在的目标不是展示一堆预测卡片，而是建立闭环：

```text
赛事接入
  -> 盘口和赔率读取
  -> 历史样本校验
  -> 模型推演
  -> 人工锁版
  -> 赛果回填
  -> 复盘诊断
  -> Case Base 样本沉淀
  -> 下一场相似案例调用
```

每一场比赛结束后，都应该让下一场判断更有依据。

## 目前仍然存在的不足

### 1. 外部样本仍不等于真实模型样本

外部历史样本可以补强分布判断，但它不能替代我们自己的赛前锁版样本。

真正有价值的样本，必须来自：

```text
赛前模型推演 -> 锁版 -> 赛后回填 -> 复盘诊断
```

只有这条链路完整，才能评估模型是否真的判断对了。

### 2. 部分联赛还缺少盘口级历史样本

有些联赛目前只有赛果画像，盘口样本还不够完整。

这意味着系统可以判断这个联赛的进球分布、平局率和常见比分，但对让球、大小球、赔率低位的历史校验还不够强。

后续需要继续补：

- 韩职历史盘口
- 瑞超更完整盘口
- 日职、挪超、丹超等联赛盘口
- 非五大联赛的赔率变化数据

### 3. 赛事详情页已经统一，但未锁版内容还只是预筛

现在所有赛事都能进入详情页。

但如果还没有模型推演，页面只会显示 `待推演 / 待锁版` 和盘口预筛信息。这是正确的，但也说明这些比赛还没有真正进入模型资产库。

后续需要做的是：让更多比赛完成真实推演，并写入锁版记录。

### 4. 多语言还没有正式接入

当前网站仍以中文为主。

如果要面向日本、英语用户，需要新增语言选择，并把界面文案、模型状态、按钮、说明文本抽成多语言词典。

更复杂的是模型长文本翻译。锁版原文不能被覆盖，最好保留中文原文，同时新增英文和日文展示层。

### 5. 仍需要更强的自动化复盘

目前系统已经能展示复盘和样本，但下一步还要增强：

- 自动识别失败原因
- 自动归类卡盘、冷门、平局保护、进球偏差
- 自动把复盘结果写回 Case Base
- 按联赛和模型版本统计长期表现

只有复盘自动化足够强，系统才会真正越用越准。

## 官网介绍短文

Football Decision Engine 是一个面向足球比赛的赛前推演和赛后复盘系统。

它把赛程、盘口、赔率、历史样本、模型判断和赛后验证放在同一套流程里。我们不追求每场比赛都给出简单结论，而是更关注判断依据是否可追溯、赛前锁版是否真实、赛后复盘是否能沉淀为下一场可用的样本。

目前系统已经从世界杯专题扩展到多联赛赛事池。世界杯使用当前 V4 模型，其它联赛从各自 V1 开始，逐步建立独立的联赛画像、盘口样本和复盘记录。

这是一个长期模型资产项目，而不是一次性的预测页面。

今天我们把 Football Decision Engine 往前推进了一大步。

它不再只是一个世界杯预测页面，而是在变成一个多联赛、可复盘、可积累样本的足球比赛决策系统。

今天主要做了 5 件事：

1. 补齐外部历史样本，尤其是世界杯和五大联赛。
2. 修正相似案例逻辑，避免重复样本和跨联赛误匹配。

4. 修复赛事卡片详情入口，任何赛事都应该能进入详情页。
5. 明确未锁版状态：没有真实推演就显示待推演 / 待锁版，不伪装成模型结论。

这个项目的核心不是“猜中一场比赛”，而是建立一条闭环：

赛事接入 -> 盘口读取 -> 历史样本校验 -> 模型推演 -> 赛前锁版 -> 赛果回填 -> 复盘诊断 -> 样本沉淀。

长期看，真正有价值的不是单场预测，而是每一场比赛结束后，系统都能多一条可复用样本。

- 外部样本还不能替代真实锁版样本。
- 部分联赛盘口历史还不够完整。
- 多语言还没正式接入。
- 自动化复盘还需要继续增强。

但方向已经清楚：

从页面展示，走向模型资产。
从主观分析，走向样本驱动。
从单一世界杯，走向多联赛决策引擎。
